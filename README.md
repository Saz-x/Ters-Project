# 🛡️ تِرس (Ters) - خدمة الأمان المالي الذكي

[![Hackathon](https://img.shields.io/badge/Hackathon-AMAD%202026-blue.svg)](#)
[![Target Partner](https://img.shields.io/badge/Designed%20For-Alinma%20Bank-brown.svg)](#)

**تِرس** هو نظام أمان مالي مبتكر ومنصة متكاملة مصممة خصيصاً لحماية المتسوقين عبر الإنترنت من السرقة والاحتيال الرقمي. تقوم فكرة المشروع على تمكين العميل من إنشاء بطاقات دفع رقمية مؤقتة (افتراضية) عبر تطبيق الجوال، وتكون مخصصة للاستخدام لمرة واحدة فقط أو وفق شروط برمجية صارمة. بمجرد إتمام عملية الشراء والدفع، يتم إلغاء بيانات البطاقة وتدميرها فوراً وبشكل تلقائي؛ وبذلك، حتى لو تعرض موقع التسوق للاختراق وسرقة البيانات، فلن يتمكن أي مخترق من سحب ريال واحد لأن البطاقة أصبحت غير موجودة تماماً في النظام.

## 🚀 الميزات الرئيسية (Key Features)

### 1. 💳 البطاقات الافتراضية المؤقتة (Virtual Disposable Cards)
إصدار فوري لأرقام بطاقات وتواريخ انتهاء وهمية وصالحة للاستخدام لمرة واحدة أو لفترة زمنية قصيرة جداً (مثل 24 ساعة فقط)، مما يمنع استخدامها على المدى الطويل.

### 2. 🔄 الرمز الأمني الديناميكي (Dynamic CVV)
تغيير الرمز الأمني الثلاثي (CVV) الموجود خلف البطاقة تلقائياً كل بضع دقائق عبر خوارزميات تشفير متزامنة. هذا الإجراء يجعل البيانات المسربة أو المنسوخة بلا قيمة فوراً بعد مرور وقت قصير.

### 3. 💻 البطاقات المبرمجة (Cards-as-Code)
ضبط شروط وأحكام البطاقة برمجياً عند التوليد. تنتهي البطاقة وتُلغى صلاحيتها ذاتياً فور تحقق الشرط المحدد (مثل: إتمام عملية شراء واحدة بنجاح)، لتتحول قيمتها وحدها الائتماني إلى صفر بشكل آلي تماماً.

### 4. ⚙️ محرك القواعد الذكي وضبط الحدود (Micro-Rule Engine)
تقنية تتيح للعميل تحديد سقف مالي دقيق وصارم لكل بطاقة (مثلاً: 50 ريالاً فقط لطلب محدد). يمنع هذا المحرك الذكي سحب أي مبلغ إضافي، ويقوم بإلغاء البطاقة فوراً وبشكل تلقائي إذا تم رصد أي محاولة لتجاوز هذا السقف.

### 5. 🛑 البطاقات أحادية الاستخدام (Disposable Cards)
توليد بطاقة رقمية مخصصة لمعاملة مالية وحيدة. وبمجرد سحب المبلغ المخصص، تذوب البطاقة رقمياً وتنتهي صلاحيتها بالكامل، ولا يمكن إعادة تنشيطها أو استخدامها تحت أي ظرف.

## ⚙️ آلية العمل (How It Works)

1. **الطلب والإصدار:** يقوم المستخدم بطلب إصدار بطاقة افتراضية من خلال التطبيق، ويحدد القيود المطلوبة (مثل الحد المالي الأقصى وشروط الاستخدام).
2. **التوليد الآمن:** تقوم خدمة **تِرس** بتوليد بيانات البطاقة (رقم البطاقة، تاريخ الانتهاء، والـ CVV الديناميكي الأول) عبر محرك القواعد الذكي.
3. **التمرير والمطابقة:** عند استخدام البطاقة في موقع التسوق، يقوم المحرك بـالتحقق الفوري من القيود والرمز الأمني النشط في تلك الدقيقة.
4. **التدمير الذاتي:** فور اكتمال عملية الخصم بنجاح، يتم إنهاء صلاحية البطاقة ومحو بياناتها من الأنظمة النشطة فوراً، لتبقى الحسابات الأساسية في أمان تام.

## 🛠️ البنية التقنية (Tech Stack)

- **تطبيق الجوال (Frontend):** مبني باستخدام إطار عمل **Flutter** لتوفير واجهة مستخدم سلسة وتجربة متكاملة متوافقة مع الهوية البصرية لـ مصرف الإنماء.
- **الخدمات الخلفية (Backend):** تطوير محرك القواعد الذكي والخدمات الأمنية باستخدام لغات **Python / Java** لضمان سرعة الاستجابة والدقة العالية في معالجة القيود برمجياً.
- **الأمان والتشفير:** الاعتماد على بروتوكولات تشفير متقدمة وخوارزميات إدارة الهوية الرقمية لتوليد الرموز الأمنية المتغيرة.

## 🚀 التشغيل والتثبيت المحلي (Setup & Installation)

### 📋 المتطلبات المسبقة (Prerequisites)
- **Frontend:** Flutter SDK (Latest Stable Version) & Dart
- **Backend:** Python 3.10+ & Pip package manager

---

### 🛠️ خطوات التشغيل (Execution Steps)

#### 1. استنساخ المستودع (Clone the Repository)
```bash
git clone [https://github.com/Saz-x/Repository.git](https://github.com/Saz-x/Repository.git)
cd ters
   ```
# الانتقال لمجلد الباكيند
cd Backend

# تثبيت مكتبات التشفير والـ API المطلوبة
python -m pip install fastapi uvicorn pyotp cryptography

# تشغيل خادم SafePay المحلي
uvicorn main:app --reload
# الانتقال لمجلد الفلاتر
cd ../Frontend

# تحميل حزم ومكتبات Flutter الاعتمادية
flutter pub get

# تشغيل التطبيق على المحاكي (Emulator) أو الهاتف المحمول
flutter run
